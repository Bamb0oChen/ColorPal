## 一、视觉规范在项目中的定位

依据 [ColorPal 技术开发规范文档](./ColorPal-技术开发规范文档.md)，前端项目结构中有以下文件与本视觉规范直接相关：

| 前端文件 | 与本规范的关系 |
|----------|---------------|
| `frontend/src/types/photo.ts` | 定义 `AnalysisResult` 类型，包含 color_id、rarity 等颜色系统字段 |
| `frontend/src/stores/photo.ts` | 管理照片/颜色收集状态 |
| `frontend/src/stores/pet.ts` | 管理小人能量，受收集成就影响 |
| `frontend/src/stores/task.ts` | 任务系统，可关联颜色收集任务 |
| `frontend/src/components/score/ColorPalette.vue` | 展示调色板色块，应用稀有度视觉规则 |
| `frontend/src/views/CollectionView.vue` | 图鉴页，展示 36 色收集网格 |

### 颜色系统设计的上下游依赖

```
AI 输出 hex → 颜色系统设计文档         → 视觉规范
              (定义 36 色/稀有度/成就)   (定义颜色怎么展示)
                      ↓                        ↓
              后端存入数据库              前端渲染 UI
              (color_id/rarity)          (色块/星级/动画)
```

---

### 1.1 核心理念

- **内容即颜色**：App 的视觉主角是用户收集到的颜色本身，界面是展示颜色的画框
- **稀有度可见**：用户一眼就能分辨一个颜色是常见还是传说
- **收集有反馈**：收集到新颜色要有"获得感"，收集进度要清晰可见

### 1.2 两个层次的颜色

| 层次 | 指什么 | 定位 | 设计原则 |
|------|--------|------|----------|
| **界面色** | App 本身的 UI 颜色（背景、文字、按钮） | 画框 | 低调、克制、统一，不抢戏 |
| **收集色** | 用户拍照收集到的 36 种颜色 | 画的内容 | 鲜艳、准确、可辨识，有稀有度区分 |

> 界面色的具体配色由前端视觉规范定义。本文档仅描述收集色的视觉呈现规则。

## 二、收集色视觉呈现规则

### 2.1 色块展示规范

收集色在 App 中以色块形式展示（图鉴页、结果页、地图标记等场景）。

| 属性 | 规则 |
|------|------|
| **形状** | 正方形（圆角可选，圆角率不超过 20%） |
| **填充** | 该颜色的准确 HEX 色值 |
| **边界** | 必须与背景有明确区分，最小对比度 3:1 |
| **尺寸** | 在不同场景下等比例缩放，保持宽高比 1:1 |

### 2.2 未收集状态

未收集到的颜色不要完全隐藏，而是显示为"空位"，让用户知道这里有一个待收集的颜色：

- 色块区域显示为浅灰色（#E8E8E8 或更浅）
- 中心显示 "?" 符号
- 边框为虚线
- 色块下方显示颜色名称（灰色文字）

### 2.3 收集状态

收集到的色块：
- 填充该颜色的准确色值
- 边框颜色跟随稀有度（见第三章）
- 可选：右上角显示稀有度等级标记

## 三、稀有度视觉呈现规则

### 3.1 稀有度视觉层级

| 稀有度 | 色块边框 | 星级标记 | 辅助视觉 | 用户感受 |
|--------|---------|----------|----------|----------|
| **常见** | 细线 1px，灰色 #B2BEC3 | 无 | 无 | 基础款 |
| **稀有** | 中等线 1.5px，蓝色 #74B9FF | ★ 一颗实心蓝星 | 轻微发光 | 有点特别 |
| **史诗** | 中等线 1.5px，紫色 #A29BFE | ★★ 两颗实心紫星 | 中等发光 | 很珍贵 |
| **传说** | 粗线 2px，金色 #FFD700 | ★★★ 三颗实心金星 | 金色光晕 | 终极稀有 |

### 3.2 星级标记规则

- 星级标记位于色块的右上角
- 星的大小与色块尺寸成比例：大色块星大，小色块星等比例缩小
- 星级颜色与稀有度标识色一致
- 常见颜色不显示星级

### 3.3 稀有度标识色

这些是稀有度本身的视觉颜色，用于徽章、标签等辅助展示：

| 稀有度 | 标识色 | 色值 | 说明 |
|--------|--------|------|------|
| 常见 | ■ 灰 | #B2BEC3 | 中性、不抢眼 |
| 稀有 | ■ 蓝 | #74B9FF | 明亮、清晰 |
| 史诗 | ■ 紫 | #A29BFE | 深邃、高级感 |
| 传说 | ■ 金 | #FFD700 | 奢华、最高级 |

## 四、成就视觉呈现规则

### 4.1 成就分类的视觉区分

| 成就类型 | 标识色 | 视觉符号 | 使用场景 |
|----------|--------|----------|----------|
| 收集类 | #6C5CE7 紫 | 🏆 奖杯 + 色系色块 | 图鉴收集相关 |
| 稀有度类 | #FFD700 金 | 👑 皇冠 + 星星 | 稀有度挑战 |
| 特殊组合 | #00B894 薄荷绿 | ✦ 星芒 + 锁链 | 组合条件 |
| 地点/时间 | #74B9FF 蓝 | 📍 定位针 + 日历 | 位置/时间相关 |

### 4.2 成就状态视觉规则

| 状态 | 视觉呈现 |
|------|----------|
| **已完成** | 彩色标识色 + 绿色 ✓ 对勾标记 + 卡片正常显示 |
| **未完成** | 灰色调 + 显示进度 "3/6" + 整体半透明 |
| **锁定** | 纯灰色 + 🔒 锁图标（仅限有条件前置的成就） |

### 4.3 成就进度视觉规则

- 进度条使用该成就分类的标识色
- 进度数值使用 "已收集数/目标数" 格式
- 进度达到 100% 时自动变为绿色 #00B894

### 4.4 成就达成反馈

当用户达成一个成就时，视觉反馈应包括：

1. **成就横幅**：从顶部或底部滑入，展示成就图标 + 名称
2. **标识色突出**：横幅使用该成就分类的标识色作为主色调
3. **进度更新**：相关进度条更新到最新状态
4. **多成就处理**：一次触发多个成就时，按 稀有度 > 收集 > 组合 > 地点 优先级依次展示

## 五、颜色路径 — 用户的收集旅程

### 5.1 收集路径总览

用户的收集路径按稀有度从低到高自然推进，对应成就系统的层级：

```
阶段一：色彩入门
  └─ 目标：收集常见色（18 种）
  └─ 关键成就：新手猎色、各色系基础收集

阶段二：深度探索
  └─ 目标：收集稀有色（11 种）
  └─ 关键成就：稀有发现者、三原色、色彩组合

阶段三：高阶猎手
  └─ 目标：收集史诗色（5 种）
  └─ 关键成就：史诗猎手、彩虹之路、冷暖对决

阶段四：终极收藏家
  └─ 目标：收集传说色 + 全部 36 色
  └─ 关键成就：传说之眼、全色谱大师
```

### 5.2 色系路径 — 用户的颜色探索地图

建议在地图页按色系展示热力图，鼓励用户探索不同色系：

```
红色系 ─── 城市建筑、室内（最容易在室内拍到）
橙色系 ─── 黄昏、食物（时间性较强）
黄色系 ─── 白天、阳光场景
绿色系 ─── 公园、自然（季节性强）
蓝色系 ─── 天空、水面、城市夜景
紫色系 ─── 花卉、灯光（最难自然拍到）
无彩色 ─── 极简场景、夜晚、金属物
```

### 5.3 收集难度参考

| 难度 | 色系 | 说明 |
|------|------|------|
| ★☆☆ | 红色系、橙色系 | 日常环境常见 |
| ★★☆ | 黄色系、蓝色系 | 需要留意 |
| ★★★ | 绿色系、紫色系 | 需特定场景 |
| ★★★★ | 无彩色 | 需要寻找纯色场景 |
| ★★★★★ | 全收集 | 长期目标 |

## 七、颜色准确性保障

### 7.1 颜色校准原则

- 36 种标准色的 HEX 色值一旦确定，不应随意修改
- 如需调整，必须同时更新 AI 识别范围和前端色值
- 每种颜色的 HEX 值是"官方色"，AI 识别输出和前端展示必须使用同一色值

### 7.2 色准检查清单

- [ ] 36 种颜色都有唯一的中文名、英文标识、HEX 色值
- [ ] 每种颜色都定义了 AI 识别色值范围
- [ ] 稀有度等级已分配且分布合理
- [ ] 每种颜色归属到 7 大色系之一
- [ ] 每种颜色的 ID 编码符合命名规范
- [ ] 成就系统的条件与颜色 ID 一一对应
- [ ] 颜色编码与接口文档中的字段名一致
- [ ] AI Prompt 中的 36 色列表与定义表一致
