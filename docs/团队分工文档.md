# ColorPal 团队分工文档

> **团队规模**：4 人
> **项目周期**：25 小时
> **并行产出**：产品 MVP + 路演海报
> **分工方式**：按功能模块垂直切分（每人前后端全栈）
> **日期**：2026.05

---

## 1. 分工原则

按功能模块切分，**每人拥有一个完整的前后端垂直切片**，互不阻塞：

```
模块 A ── 拍照评分 ──→ 模块 B ── 宠物任务 ──→ 模块 C ── 地图收集
  │                      │                      │
  └── 写入「照片数据」    └── 写入「任务数据」    └── 展示「所有数据」
                                                      │
                                        模块 D ── 产品设计（非代码）
```

**好处**：
- 无人阻塞：每人做完自己的模块就能跑，不用等其他人
- 接口简单：模块间通过数据库共享数据，不需要联调 API
- 风险隔离：某个模块做不完不影响其他模块演示

---

| 模块 | 人员 | 功能 | 前端 | 后端 | 可独立运行 |
|------|------|------|------|------|-----------|

## 3. 详细职责

| 前端 | 后端 | 工时 |

| 评分结果卡片（分数 + 色板 + 评语） | `services/scorer.py` 规则评分兜底 | 2h |
| 首页布局 | 后处理校验 | 1.5h |
| Loading / 错误 / 空状态 | — | 1h |
| **小计** | | **9h** |

**独立可跑**：做完即可演示「拍照 → AI 分析 → 评分展示」

### 模块 B：宠物任务（成员 2）

| 前端 | 后端 | 工时 |
|------|------|------|
| `PetDisplay.vue` 小人 SVG 展示 + 换色 | `GET /user/profile` 用户 + 小人状态 | 2h |

| `TaskBubble.vue` 任务气泡 + 完成 | `GET /task/current` 当前任务 | 1.5h |

## 5. 协作约定

- **A、B、C 从第 0 小时并行开工**，无前置依赖
- B 用 mock 数据开发，C 用 seed 数据开发
- 模块间通过数据库共享，不互相调 API
- `photo_records` 表结构是核心契约，A 写入、C 读取
- 站会：每天 2 次，每次 5 分钟

---

## 6. 风险应对

| 风险 | 应对 |
|------|------|
| A 做不完 → 核心拍评分断裂 | B 协助 A；B 的任务系统可以 mock |
| B 做不完 → 小人没反馈 | 评分结果页直接展示分数，跳过小人 |
| C 做不完 → 展示页不全 | 砍地图/图鉴，首页+评分结果页够演示 |
| GPT-4V 不可用 | 规则评分自动降级 |
