## 一、颜色系统在项目架构中的位置

依据 [ColorPal 技术开发规范文档](./ColorPal-技术开发规范文档.md) 定义的架构，颜色系统在以下环节起作用：

```
用户拍照 → Vue 3 前端 → FastAPI → Qwen/DeepSeek 视觉分析
                                     │
                              [颜色系统在这里]
                              AI 按 36 色分类输出
                                     │
                              FastAPI 映射到标准色
                                     │
                              ┌──────┴──────┐
                              ↓              ↓
                          SQLite 入库    返回给前端
                          (颜色ID/稀有度)  (色块展示/成就)
```

### 1.1 各层的颜色数据流转

| 架构层 | 对应的颜色系统数据 | 技术开发规范中的对应文件 |
|--------|------------------|------------------------|
| **视觉模型分析** | 按 36 色分类识别主色，输出 color_id | `gateway/app/services/ai_analyzer.py` |
| **FastAPI 网关** | 将 AI 输出映射到标准色，持久化照片与用户状态 | `gateway/app/routers/photo.py` |
| **SQLite 数据库** | 存储颜色 ID、稀有度、收集状态、成就进度 | `gateway/app/models/photo.py` |
| **Vue 前端** | 展示色块、稀有度徽章、成就状态 | `frontend/src/stores/photo.ts`、`frontend/src/types/photo.ts` |

### 1.2 颜色系统影响的接口链路

```
前端 POST /api/v1/photo/analyze
  → FastAPI 调用 Qwen/DeepSeek 视觉模型
  → AI 返回 dominant_color + palette
  → FastAPI 匹配 36 色 → 得到 color_id + rarity
  → FastAPI 入库，更新收集计数 + 检查成就
  → 返回 { color_id, rarity, is_new, achievements_unlocked }
  → 前端展示色块 + 稀有度 + 成就弹窗
```

### 1.3 颜色分类总则

ColorPal 将自然界的颜色归纳为 **7 大色系、36 种标准色**。每种颜色有：

| 属性 | 说明 |
|------|------|
| **中文名** | 用户端展示的名称 |
| **英文标识** | 系统内部标识，AI 输出用 |
| **色值** | HEX 格式，精确到 6 位 |
| **色系归属** | 红/橙/黄/绿/蓝/紫/无彩 |
| **稀有度** | 常见/稀有/史诗/传说 |
| **识别范围** | AI 判定为该色的色值区间 |

## 二、稀有度系统

### 2.1 稀有度定义与分布

| 稀有度 | 标识色 | 数量 | 占比 | 识别条件 |
|--------|--------|------|------|----------|
| **常见** | #B2BEC3 | 18 种 | 50% | 日常环境中高频出现的颜色 |
| **稀有** | #74B9FF | 11 种 | 31% | 需要特定场景或刻意寻找 |
| **史诗** | #A29BFE | 5 种 | 14% | 深色/高饱和/特定时间光线 |
| **传说** | #FFD700 | 2 种 | 5% | 金属质感 + 特定色值双条件 |

### 2.2 稀有度判定逻辑

AI 识别到一种颜色后，按以下流程判定稀有度：

```
AI 输出色值 → 匹配 36 色中最接近的标准色
           → 读取该标准色的稀有度等级
           → 如果同时匹配多个（如金黄与金色相冲突）→ 优先判定为传说
```

### 2.3 稀有度视觉标识

| 稀有度 | 色块边框 | 徽章 | 说明 |
|--------|---------|------|------|
| 常见 | 灰色实线 | 无 | 基础展示 |
| 稀有 | 蓝色实线 | ★ 一颗星 | 轻微视觉强调 |
| 史诗 | 紫色实线 | ★★ 两颗星 | 明显视觉突出 |
| 传说 | 金色实线 | ★★★ 三颗星 | 最高等级视觉待遇 |

> 注：以上为颜色系统的视觉规则，具体 CSS 实现由前端团队在视觉规范中定义。

## 三、AI 颜色识别规则

### 3.1 识别流程

```
用户拍照 → FastAPI → Qwen/DeepSeek 视觉分析
                   → 输出 dominant_color (#FF6B6B)
                   → 匹配 36 色中最接近的标准色 → 粉红 (Pink)
                   → 读取稀有度 → 常见
                   → 返回 { color_id: "pink", rarity: "common", ... }
```

### 3.2 匹配规则

1. **六边形色距匹配**：将 AI 输出的色值转为 HSL，计算与 36 种标准色在色相环上的距离
2. **就近归入**：取距离最近的标准色作为结果
3. **阈值排除**：与所有标准色的距离均超过阈值时，归入"未分类"，不计入收集
4. **金属色特殊规则**：银和金需要额外检测图片中是否有金属质感（高光、反射），仅凭色值不足以判定

### 3.3 多主色场景

当一张照片中有多个主色时：
- 取面积最大的颜色为主色，计入 36 色收集
- 辅助色在结果页展示调色板，但不单独计入收集进度
- 调色板中的辅助色最多展示 5 个

### 3.4 AI Prompt 建议

```text
你是一位色彩分类专家。分析照片的主色，输出 JSON：
{
  "dominant_color_hex": "#FF6B6B",   // 主色的十六进制色值
  "palette": ["#FF6B6B", "#4ECDC4"], // 调色板（最多5色）
  "color_name": "粉红",              // 匹配到的标准色名称
  "confidence": 0.92                // 匹配置信度
}

匹配规则：
- 将 dominant_color 与以下 36 种标准色比较，取最接近的
  （此处内嵌 36 种颜色的色值列表）
- 如果色值接近金色 (#FFD700) 且图片有金属光泽，优先判定为"金"
- 如果无法匹配任何标准色（色距过大），color_name 返回 "unknown"
```

## 五、颜色收集与成就关联规则

### 5.1 收集触发流程

```
用户拍照 → AI 分析 → 匹配 36 色成功？
                    → 否 → 不计入收集，仅显示评分
                    → 是 → 该色首次收集？
                          → 是 → 记录收集 +1
                                → 检查是否触发成就
                                → 弹出"新颜色收集"通知
                                → 如有成就达成 → 弹出"成就解锁"通知
                          → 否 → 仅显示评分，不做收集处理
```

### 5.2 成就判定时机

- **即时判定**：每次新颜色入库时，检查所有与该颜色相关的成就
- **不重复触发**：已完成的成就不再触发
- **批量判定**：如果一次入库同时满足多个成就，按稀有度类 > 收集类 > 特殊组合 > 地点/时间的优先级排序展示

### 5.3 颜色稀有度与成就的关系

```
用户收集到新颜色 → 该颜色是传说级？
                → 是 → 直接触发"传说之眼"成就进度更新
                → 否 → 仅计入对应色系和稀有度计数
                
"全色谱大师"成就的隐含路径：
  常见（18 种）→ 探索基础 → 新手猎色
     ↓
  稀有（11 种）→ 深入探索 → 稀有发现者
     ↓
  史诗（5 种）→ 高阶收集 → 史诗猎手
     ↓
  传说（2 种）→ 终极目标 → 传说之眼 → 全色谱大师
```

### 5.4 成就奖励规则

| 成就类型 | 奖励 |
|----------|------|
| 收集类 | 小人获得该色系的主题装饰 + 能量 +50 |
| 稀有度类 | 小人升级加速 + 对应稀有度的色块徽章 |
| 特殊组合 | 限定称号 + 能量 +100 |
| 传说之眼 / 全色谱大师 | 限定小人皮肤 + 全服公告 |

## 六、颜色系统的扩展性

### 6.1 未来可扩展方向

| 方向 | 说明 | 影响范围 |
|------|------|----------|
| 增加标准色 | 从 36 色扩展到 48/60 色 | AI 识别范围 + 成就条件 + 入库逻辑 |
| 季节限定色 | 仅在特定时间段可收集 | 时间成就 + 收集池动态变化 |
| 用户自定义色 | 用户上传色值自行命名 | 独立于 36 色体系，不纳入成就 |
| 地区限定色 | 特定城市/地标专属颜色 | GPS 判定 + 图鉴特殊槽位 |

### 6.2 36 色 ID 编码规范

建议后端使用以下 ID 编码，保持前后端一致：

```
格式：{色系缩写}_{英文小写}
示例：red_pink, blue_sky, gray_black, gray_silver

色系缩写：red / orange / yellow / green / blue / purple / gray
```

### 6.3 数据接口建议

新增颜色时，后端返回字段建议：

```json
{
  "color_id": "red_pink",
  "color_name": "粉红",
  "hex": "#FFC0CB",
  "family": "red",
  "rarity": "common",
  "is_new": true,
  "collection_progress": {
    "family_collected": 3,
    "family_total": 6,
    "total_collected": 12,
    "total_all": 36
  },
  "achievements_unlocked": []
}
```
