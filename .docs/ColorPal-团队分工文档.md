# ColorPal 团队分工文档

> **团队规模**：4 人
> **项目周期**：25 小时（集中开发）
> **并行产出**：产品 MVP + 路演海报
> **日期**：2026.05

---

## 1. 分工原则

- 按**功能模块**切分，避免两个人改同一段代码
- 前端开发是**关键路径**——影响 App 进度和海报产出
- 路演海报由产品/设计主导，但**依赖前端产出视觉素材**，需规划好交付节点
- 每位成员有明确的「主责」和「协作」任务

---

## 2. 角色总览

| 角色 | 人员 | 主责领域 | 涉及代码/产出 |
|------|------|----------|---------------|
| **A · 前端开发** | 成员 1 | App UI、相机、小人动画、海报视觉素材产出 | 前端所有代码 |
| **B · 算法/AI** | 成员 2 | 视觉模型集成、Prompt、评分规则、任务生成 | AI 云函数、Prompt 模板 |
| **C · 后端开发** | 成员 3 | 云函数、数据库、用户状态、任务/图鉴/地图 API | 后端云函数、数据模型 |
| **D · 产品/设计** | 成员 4 | 产品文档、路演海报、演示视频、GitHub、团队协调 | 文档、海报设计稿、PRD |

### B · 算法/AI

| 申请视觉模型 API Key + 单张图片调用测试 | 1h | — | API 可用确认 |

**关键路径说明**：

## 6. 沟通规则

- **每 2 小时站会**：同步进度、暴露阻塞
- **一方阻塞立刻喊人**：B 的 API 没调好，A 必须知道并切换到可并行的工作
- **不得等待**：如果前置依赖未完成，先做自己可独立推进的任务
- **共用素材**：所有截图/素材统一放在共享文件夹（建议飞书/网盘）

---

## 7. 风险：如果有人掉链子

| 场景 | 应对 |
|------|------|
| A（前端）进度落后 | C（后端）协助前端写简单页面；D 先用设计稿截图做海报 |
| B（AI）API 调不通 | 切换到纯规则评分方案，AI 评分作为增强选项 |
| C（后端）做不完 | 砍掉图鉴 API（前端硬编码演示数据）；砍掉地图 API（用假数据） |
| D（产品）海报做不完 | 用 GitHub 封面 + README + 简单拼图代替 |
| 多人同时阻塞 | 优先保核心闭环（拍照→评分→小人），其他全砍 |
