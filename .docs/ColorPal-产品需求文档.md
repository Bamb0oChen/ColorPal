## 1. 产品概述

### 1.1 一句话定义

ColorPal 是一个「养小人」的趣味拍照应用。用户拍照后，AI 提取照片中的颜色并评分，虚拟小人「吃掉」颜色后成长进化，形成持续探索的正向循环。

### 1.2 产品 Slogan

> 喂它颜色，陪它成长，用新的方式看世界。

### 1.3 核心价值主张

我们不是在做一个拍照工具，而是在创造一个**用色彩连接观察与成长的新体验**。ColorPal 用游戏化的方式解决「人们缺乏观察动机」的问题——让每一次拍照都成为喂养小人的互动，让每一天的出行都变成一场色彩探索。

## 2. 市场背景与机会

### 2.1 用户洞察

| 痛点 | 机会 |
|------|------|
| 人们每天看手机，很少真正「观察」身边的世界 | 需要一个理由去抬头看、去发现 |
| 拍照只为了记录，缺少持续的创作动机 | 游戏化的养成机制可以驱动探索行为 |
| 色彩/设计/审美类内容受欢迎，但缺少低门槛的日常练习工具 | Colorwalk 本身就是一种被验证的创作方式 |
| 社交平台上色卡/配色方案内容传播力强 | 视觉驱动的产品天然适合抖音裂变 |

### 2.2 目标用户画像

- **核心用户**：18–30 岁，对审美/色彩/设计有兴趣的年轻人
- **次核心用户**：摄影爱好者、设计学生、日常喜欢 Citywalk 的人群
- **传播用户**：抖音/小红书活跃用户，愿意分享创意内容

## 3. 核心玩法设计

### 3.1 总览流程图

```
[拍照] → [AI 提取颜色 + 评分] → [小人吸收/成长] → [解锁图鉴/成就] → [触发新任务]
   ↑                                                              ↓
   └────────────────── [地图记录 Colorwalk 路线] ←─────────────────┘
                           ↓
                    [社区/组队/分享小人]
```

### 3.2 玩法一：喂食与评分（核心闭环）

**流程**：
1. 用户拍照（或从相册选择）
2. AI 提取照片的**主色调 + 配色方案**
3. 基于色彩理论输出 **评分（0–100分）** + 一句评价
   - 「这个蓝橙互补色很高级」→ 85 分
   - 「颜色太灰了，缺少对比」→ 42 分
4. 小人「吃掉」颜色 → 获得能量值 → 小人外观/心情变化

**设计要点**：
- 评分标准基于客观色彩理论（互补色、对比度、饱和度等），非随机
- 每张照片都有结果反馈，不存在「白拍」
- 即时反馈驱动用户尝试「能不能拍出更高分」

**技术路线**：
- MVP 阶段：调用 Qwen / DeepSeek 视觉模型，结合简单色彩规则
- 后期：可训练专用色彩评分模型

### 3.3 玩法二：任务系统（驱动持续探索）

**任务类型**：

| 类型 | 示例 | 目的 |
|------|------|------|
| 颜色收集 | 拍一张红色的东西 | 引导基础探索 |
| 组合挑战 | 拍一组对比色 | 提升审美感知 |
| 连续任务 | 连续 3 天拍暖色调 | 培养习惯 |
| 地点相关 | 在公园拍一张绿色 | 结合地图探索 |
| 时间相关 | 拍一张日落时分的照片 | 增加仪式感 |
| 社区任务 | 模仿本周最佳色板 | 社交驱动 |

**任务奖励**：
- 双倍能量
- 特殊色卡（图鉴解锁）
- 小人进化道具
- 成就徽章

### 3.4 玩法三：Colorwalk 路径规划（延展玩法）

**场景**：用户要去一个地方（旅游、散步、Citywalk），小人为其「定制一条有色彩看点的路线」。

**机制**：
1. 用户输入目的地/区域
2. 小人基于历史数据 + 地图 POI，生成一条 Colorwalk 路线
3. 用户边走边拍 → 喂给小人 → 路线完成度记录

**与主体玩法的融合**：
- 路径规划是**任务的延伸**，不是独立功能
- 完成一条路线 = 解锁特殊成就 + 地图徽章
- 路线可分享到社区，其他人可以「重走」

> **MVP 阶段简化方案**：不做实时路径规划，先做 3–5 个预设地点任务。

## 4. 成长与收集系统

### 4.1 图鉴系统

| 图鉴类型 | 说明 |
|----------|------|
| 色卡图鉴 | 每解锁一种颜色/配色组合，收录进图鉴 |
| 地点图鉴 | 在不同类型地点拍的照片（公园/街道/美术馆/咖啡店） |
| 特殊图鉴 | 节日限定、季节限定、稀有配色 |

### 4.2 成就系统

| 成就类型 | 示例 |
|----------|------|
| 数量型 | 拍满 100 张照片 |
| 连续型 | 连续 7 天拍照 |
| 挑战型 | 拍出一张 90 分以上的照片 |
| 探索型 | 走完 5 条 Colorwalk 路线 |
| 社交型 | 被 10 个人点赞你的色板 |

### 4.3 进化系统

- 小人有多阶段形态：**幼年 → 成长 → 成熟 → 稀有形态**
- 进化条件：特定颜色组合 + 能量值 + 完成特定成就
- 不同形态对应不同的能力/任务类型/外观
- 进化后解锁新的图鉴区域

> **MVP 阶段**：用 Live2D 实现基础形态、表情和动作；阶段差异先用同一模型的 motion/expression/tint 参数驱动，后续再替换不同阶段模型资源。

## 5. 地图与空间记录

### 5.1 功能说明

- 每次拍照自动记录**地理位置 + 时间 + 提取的颜色**
- 在地图上生成一个**彩色标记点**（颜色即拍到的颜色）
- 用户可以回看自己的色彩足迹

### 5.2 MVP 简化方案

| 功能 | MVP 做不做 |
|------|-----------|
| 拍照时记录地理位置 | ✅ 做 |
| 地图上生成彩色标记点 | ✅ 做（简单版） |
| 路线回放动画 | ❌ 砍 |
| 完整的 Colorwalk 轨迹 | ❌ 砍 |

### 5.3 用户价值

- 把「拍照」变成「在地图上留下颜色足迹」
- 强化「探索」感，不是随机拍照
- 可视化呈现用户的色彩日记

## 6. 社区与社交系统

> **MVP 阶段仅保留分享功能，其余全砍。**

### 6.1 个人主页

- 展示小人（当前形态、心情、名字）
- 色卡图鉴完成度
- 成就徽章墙
- 地图彩色足迹（可隐藏）

### 6.2 社交功能

| 功能 | 说明 | MVP 做不做 |
|------|------|-----------|
| 分享小人 | 生成一张「小人名片」分享图 | ✅ 做 |
| 分享色板 | 拍出高分照片可一键生成「今日色卡」 | ✅ 做 |
| 色板广场 | 用户上传自己的本周色板 | ❌ 砍 |
| 组队任务 | 和好友一起完成任务 | ❌ 砍 |
| 参观他人 | 查看别人的小人和图鉴 | ❌ 砍 |
| 路线分享 | 分享 Colorwalk 路线 | ❌ 砍 |

### 6.3 传播设计

- 高分照片 → 一键生成「今日色卡」→ 适合发抖音/朋友圈
- 小人进化 → 生成「进化时刻」分享图
- 分享图包含二维码/口令，引导下载 App

## 7. 用户旅程（完整闭环）

```
用户下载 App → 领养一只灰白色的小人
       ↓
拍照 → AI 评分 → 小人吃掉颜色 → 变色/变开心
       ↓
小人发布任务：「我想看蓝色的东西」
       ↓
用户出门/翻相册 → 拍蓝色 → 获得双倍能量
       ↓
能量够了 → 小人进化 → 解锁新图鉴
       ↓
用户查看地图，看到自己走过的彩色路线
       ↓
把小人分享给朋友 → 组队完成任务（后期）
       ↓
用户养成习惯：每天出门「遛小人」
```

## 8. MVP 范围与优先级

### 8.1 优先级定义

- **P0**：必须做，否则产品不成立
- **P1**：尽量做，体验加分
- **P2**：有时间再做，展示潜力

### 8.2 MVP 功能清单

| 优先级 | 模块 | 说明 |
|--------|------|------|
| P0 | 拍照/选图 | 用户拍照或从相册选图 |
| P0 | 颜色提取 | 调用视觉模型，输出主色调 + 配色方案 |
| P0 | 颜色评分 | 基于色彩理论输出 0–100 分 + 一句话评价 |
| P0 | 小人状态 | 能量槽 + 小人外观随能量变化 |
| P1 | 图鉴系统 | 记录用户解锁过的颜色/色卡 |
| P1 | 任务系统 | 小人主动发任务，完成给奖励 |
| P1 | 地图标记 | 拍照位置 + 彩色标记点展示 |
| P1 | 分享功能 | 生成今日色板/小人状态分享图 |
| P2 | 进化系统 | 能量攒够 + 特定条件 → 形态升级 |
| P2 | 社区功能 | 色板广场、组队、路线分享 |

### 8.3 开发顺序（25 小时计划）

| 顺序 | 功能 | 预估时间 |
|------|------|----------|
| 1 | 前端：拍照/选图界面 | 2h |
| 2 | 后端：调用视觉模型 API，返回颜色+评分 | 3h |
| 3 | 前端：展示评分结果 + 小人动画反馈 | 3h |
| 4 | 后端：用户状态存储（能量、小人形态） | 2h |
| 5 | 前端：小人状态展示（能量槽 + 外观变化） | 3h |
| 6 | 任务系统：简单规则任务（3 个模板） | 3h |
| 7 | 图鉴：记录颜色的简单展示 | 2h |
| 8 | 地图：记录位置 + 展示彩色点 | 3h |
| 9 | 分享：生成「今日色板」图片 | 2h |
| 10 | 进化：能量满后手动触发形态变化 | 2h |
| **总计** | | **25h** |

## 10. 风险与应对

| 风险 | 应对方案 |
|------|----------|
| 视觉模型 API 响应慢 | 加 loading 动画；或用更小的模型（CLIP + 规则打分）做降级方案 |
| 颜色评分主观 | 明确标注「基于经典色彩理论」，并允许用户手动调整评分权重 |
| 地图功能太复杂 | 只做点标记，不做路线连线 |
| Live2D 接入做不完 | 先接静态 Live2D 模型 + 基础 motion/expression，保留 SVG 仅作临时占位 |
| 社区功能太复杂 | 只做最简单的分享图片，其余全砍 |
| 用户新鲜感消退快 | 通过任务系统 + 进化系统保持中长期动力；后期靠社区承接 |
