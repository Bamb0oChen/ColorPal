# 小彩 Live2D 样式规范

> 版本：v0.1
> 适用范围：MVP 阶段的小彩 PetDisplay、主页、结果页、个人页展示

## 1. 模型选择

- 展示页：https://imuncle.github.io/live2d/
- 模型编号：20
- 对应资源：`model/wanko/wanko.model.json`
- 模型类型：Live2D Cubism 2
- 资源结构：
  - `wanko.moc`
  - `wanko.1024/texture_00.png`
  - `motions/idle_01.mtn` 至 `idle_04.mtn`
  - `motions/shake_01.mtn`、`shake_02.mtn`
  - `motions/touch_01.mtn` 至 `touch_06.mtn`

选择理由：这个模型是白色软团主体 + 红色木碗/器皿组合。白色主体适合承接
`pet.color` 的染色与光晕变化，碗/器皿可以自然解释为小彩“吃颜色、存颜色”的
容器，比直接使用人形看板娘更符合 ColorPal 的“色彩伙伴”定位。

## 2. 角色设定

小彩不是普通宠物，也不是看板娘，而是住在调色碗里的小色彩伙伴。用户拍照后，
小彩会把照片里的主色调“吃掉”，把颜色储存在碗里，逐渐长出更丰富的色滴、光环
和色卡装饰。

核心关键词：

- 亲近：圆润、轻盈、像一个可以陪用户出门的小伙伴
- 色彩感：用光晕、色滴、色卡、调色环表达成长
- 观察感：反馈应该像“我看到了你拍的颜色”，而不是单纯数值变化
- 仪式感：进化必须由用户手动触发，不自动播放进化

## 3. 视觉结构

MVP 不直接改 Live2D 原始贴图，优先用前端叠加层完成品牌化。

推荐层级：

1. 背景层：柔和浅色背景，跟随 `pet.color` 生成 12%-18% 透明度径向光。
2. Live2D 层：渲染 `wanko` 模型，主体居中，底部保留碗的完整形状。
3. 色彩层：围绕碗口与主体周围渲染色滴、三色能量珠、调色环。
4. 状态层：显示心情、任务气泡、进化提示，不遮挡模型脸部与碗口。

布局建议：

- 主页：小彩占首屏 45%-55% 高度，拍照按钮在下方，避免把模型压成小图标。
- 结果页：小彩在评分卡旁或下方回应本次颜色，色滴使用照片 palette。
- 个人页：小彩居中展示，旁边展示 stage、energy、mood、图鉴进度。
- 地图页：不放完整模型，只用小彩头像/色滴作为地图辅助标记。

## 4. 状态映射

### 4.1 颜色

`pet.color` 是小彩的当前主色。它不改变模型本体结构，而驱动以下表现：

- 主光晕颜色
- 碗口反光颜色
- 最大的一颗色滴颜色
- 任务气泡边框色
- 进度条当前色

RGB 能量不直接用文字展示为主，优先显示为三颗能量珠：

| 能量 | 视觉 |
|------|------|
| `energy.r` | 红色珠，靠左 |
| `energy.g` | 绿色珠，靠中 |
| `energy.b` | 蓝色珠，靠右 |

能量占比越高，珠子越亮、越大，但最大尺寸不要超过最小尺寸的 1.35 倍，避免 UI 跳动。

### 4.2 心情

| `pet.mood` | 触发条件 | 动作与视觉 |
|------------|----------|------------|
| `happy` | 最近 5 张平均分 >= 70 | 快一点的 idle，光晕更亮，色滴轻微上浮 |
| `neutral` | 最近 5 张平均分 40-69 | 普通 idle，光晕稳定，色滴轻微呼吸 |
| `sad` | 最近 5 张平均分 < 40 | 慢一点的 idle，光晕降低饱和度，色滴减少 |

如果当前 Live2D runtime 暂时无法精细控制 motion speed，MVP 可以先通过外层色滴动画
和光晕透明度表达心情。

### 4.3 成长阶段

阶段不要换成完全不同角色，避免用户失去“小彩就是同一个伙伴”的连续感。

| `pet.stage` | 名称 | 视觉升级 |
|-------------|------|----------|
| 0 | 初生 | 1-2 颗色滴，低强度光晕 |
| 1 | 成长 | 3-4 颗色滴，出现三色能量珠 |
| 2 | 成熟 | 出现半透明调色环，碗口有主色反光 |
| 3 | 稀有 | 出现组合色卡与短时稀有光效 |

进化入口只在满足后端条件时展示。按钮文案建议为“让小彩进化”，点击后播放
`shake` 或更强的 `tap_body` 动作，再切换 stage 装饰。

## 5. 动作使用

`wanko.model.json` 已配置的动作组：

- `idle`：默认待机。用于主页、个人页常驻展示。
- `shake`：短反馈。用于拍照分析完成、进化前奏、高分奖励。
- `tap_body`：用户点击互动。用于摸摸小彩、任务反馈、低分安慰。

推荐事件映射：

| 事件 | 动作 | 外层反馈 |
|------|------|----------|
| 页面进入 | `idle` | 光晕淡入 |
| 分析 loading | `idle` | 色滴绕碗口慢速旋转 |
| 分析成功 | `shake` | 主色滴落入碗口 |
| 高分 >= 85 | `shake` | palette 色卡短暂展开 |
| 点击小彩 | `tap_body` | 气泡回应一句短文案 |
| 满足进化条件 | `idle` | 进化按钮轻微呼吸 |
| 手动进化 | `shake` | 调色环扩散后 stage 升级 |

## 6. 文案语气

小彩说话要像一个会被颜色喂饱的小伙伴，句子短、轻、具体。

示例：

- “我吃到一点珊瑚红。”
- “这组蓝和橙很亮，我喜欢。”
- “今天的颜色有点灰，我们再找找。”
- “能量满了，想让我长大一点吗？”

避免：

- “系统检测到颜色能量更新”
- “当前评分不足，请重新拍摄”
- “您已触发宠物进化条件”

## 7. 前端实现口径

建议新增 `PetDisplay` 组件，输入保持纯状态：

```ts
interface PetDisplayProps {
  name: string
  stage: number
  mood: 'happy' | 'neutral' | 'sad'
  color: string
  energy: {
    current: number
    max: number
    r: number
    g: number
    b: number
  }
  palette?: string[]
}
```

组件内部负责：

- 加载 Live2D `model/wanko/wanko.model.json`
- 将 `pet.color` 映射到 CSS 变量 `--pet-color`
- 将 `mood` 映射到外层动画强度
- 将 `stage` 映射到装饰数量和进化提示
- 将 `palette` 映射到结果页色滴

不要让业务页直接操作 Live2D runtime。业务页只传 `pet`、`palette`、事件回调。

## 8. 版权与上线风险

imuncle/live2d 仓库说明这些模型收集自互联网，版权归原公司/个人所有，且建议仅用于
学习、非营利网站或项目。MVP 演示可以先用第 20 个模型验证体验，但正式上线前需要：

- 确认可商用授权，或
- 重新绘制自有“小彩”模型并按相同状态映射接入，或
- 使用明确可商用的 Live2D 模型资源。

因此，代码中不要把 `wanko` 写死成不可替换方案。建议配置层命名为 `defaultPetModel`，
后续替换模型时保持 `PetDisplay` 的状态接口不变。
