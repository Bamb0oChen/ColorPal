# ColorPal 团队分工文档

> **团队规模**：4 人
> **项目周期**：25 小时（集中开发）
> **并行产出**：产品 MVP + 路演海报
> **日期**：2026.05

---

## 1. 分工原则

- 按**功能模块**切分，避免两个人改同一段代码
- 前端开发是**关键路径**——影响 App 进度和海报产出
- 路演海报由产品/设计主导，但**依赖前端产出视觉素材**，需规划好交付节点
- 每位成员有明确的「主责」和「协作」任务

---

## 2. 角色总览

| 角色 | 人员 | 主责领域 | 涉及代码/产出 |
|------|------|----------|---------------|
| **A · 前端开发** | 成员 1 | Vue 3 前端、相机/上传、小人动画、海报视觉素材产出 | `frontend/` 全部 |
| **B · 算法/AI** | 成员 2 | Qwen/DeepSeek 视觉模型集成、Prompt、HSL 兜底评分、任务匹配规则 | `backend/app/services/` 下 AI/scoring/task_matcher 模块 |
| **C · 后端开发** | 成员 3 | FastAPI 路由层、SQLAlchemy 模型、用户/照片/任务/图鉴持久化 | `backend/app/{routers,models,schemas,db.py}` |
| **D · 产品/设计** | 成员 4 | 产品文档、路演海报、演示视频、GitHub、团队协调 | 文档、海报设计稿、PRD |

> **B 与 C 共享 `backend/` 仓库**：B 写服务层（services/），C 写路由层 + ORM 模型，二者通过 Pydantic schema 对齐。提前定好 schema，就可以并行开发。

## 6. 沟通规则

- **每 2 小时站会**：同步进度、暴露阻塞
- **一方阻塞立刻喊人**：B 的 API 没调好，A 必须知道并切换到可并行的工作
- **不得等待**：如果前置依赖未完成，先做自己可独立推进的任务
- **共用素材**：所有截图/素材统一放在共享文件夹（建议飞书/网盘）

---

## 7. 风险：如果有人掉链子

| 场景 | 应对 |
|------|------|
| A（前端）进度落后 | C（后端）协助前端写简单页面（Python 转 Vue 学习成本低）；D 先用设计稿截图做海报 |
| B（AI）API 调不通 | `analyze_image()` 自动走 `fallback_score()`，无需切换代码路径 |
| C（后端）做不完 | 砍掉图鉴 / 地图 API，前端硬编码演示数据；保留拍照→评分主链路 |
| D（产品）海报做不完 | 用 GitHub 封面 + README + 简单拼图代替 |
| 多人同时阻塞 | 优先保核心闭环（拍照→评分→小人），其他全砍 |
