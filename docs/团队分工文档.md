# ColorPal 团队分工文档

> **团队规模**：4 人
> **项目周期**：25 小时（集中开发）
> **并行产出**：产品 MVP + 路演海报
> **架构**：Vue 3 → FastAPI（All-in-One）→ SQLite
> **日期**：2026.05

---

## 1. 分工原则

- 按**功能模块**切分，避免两个人改同一段代码
- **关键路径**：前端（A）最慢，决定项目交付时间
- B（AI）和 C（后端）都在 FastAPI 上工作，但**文件完全不重叠**：
  - B 改 `services/ai_analyzer.py`、`services/scorer.py`、`routers/photo.py`（AI 部分）
  - C 改 `models/`、`services/crud_*`、`routers/`（除 photo AI 外）
- 路演海报由 D 主导，**依赖 A 提供截图素材**
- **Day 1 先打通端到端最小链路**，再填业务逻辑

## 6. 沟通规则

- **每 2h 站会**：同步进度、暴露阻塞
- **一方阻塞立刻喊人**：不等人，先切到可并行的工作
- **接口契约**：以 FastAPI 的 Pydantic Schema 为准，C 定义后发给 A 和 B

---

## 7. 风险应对

| 风险 | 应对 |
|------|------|
| A（前端）进度滞后 | C 协助前端写简单页面；D 用设计稿截图做海报 |
| B（AI）API 调不通 | 切到纯规则评分，AI 评分作为增强 |
| C（后端）做不完 | 砍图鉴/地图 API，前端硬编码演示数据 |
| GPT-4V 超时 | FastAPI 自动降级到规则评分 |
| 浏览器相机不支持 | 降级到 input[type=file] |
